## 🔄 実装時の注意事項

### 1 タスク = 1 コミット原則

各タスクは最小単位で実装し、1コミットとする。

### ブランチ戦略

```
main
├── feature/todo-tags
├── feature/todo-search
├── feature/todo-sort
└── ...
```

### PR 単位

- Phase A-1: 検索機能（13タスク）
- Phase A-2: ソート機能（13タスク）
- ...

各機能を 1 PR として実装することを推奨。

### テスト必須

各機能の実装後、必ずテストを実行：

```bash
# Backend
docker compose run --rm backend npm run test

# Frontend
docker compose run --rm frontend ng test --watch=false
```

## 🚀 開始方法

1. `TODO_ADDITIONAL_FEATURES.md` で全体像を確認
2. Phase A から順に実装開始
3. 各機能の詳細ファイルを参照してタスクを実行
4. 1タスクごとにコミット
5. 1機能ごとに PR 作成
